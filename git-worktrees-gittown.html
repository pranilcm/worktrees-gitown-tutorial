git worktree

Multiple working directories,
one repository

Git Worktrees let you check out multiple branches simultaneously into separate directories — all sharing the same .git store. No more stashing, no more switching, no context loss.

Architecture — Shared .git Object Store
⬡ .git / object store
The Problem: Normally, switching branches means dropping everything you're doing. Stash your changes, checkout, do work, switch back, pop stash. It's friction that kills flow.
The Solution: Worktrees give each branch its own directory. Your main lives at ./project, your feature at ../project-feature, your hotfix at ../project-hotfix — all live, all active.

git worktree add <path> <branch>
Create a new worktree at the given path, checking out the specified branch.
git worktree list
Show all linked worktrees with their paths, HEAD commits, and branch names.
git worktree remove <path>
Remove a worktree. The branch itself is preserved; only the directory is removed.
git worktree prune
Remove stale worktree references that no longer have a valid directory on disk.

git town

High-level Git workflow,
one command

Git Town automates the tedious multi-step Git workflows that slow every team down. Create, sync, and ship branches with a single command — it handles the choreography so you don't have to.

Branch Lifecycle — Git Town Workflow
git town hack
# Create a feature branch the right way ❯ git town hack feature/payment-gateway # What it does under the hood: 1. git fetch origin 2. git checkout main 3. git rebase origin/main 4. git checkout -b feature/payment-gateway 5. git push -u origin feature/payment-gateway ✓ On branch feature/payment-gateway ✓ Up to date with origin/main ✓ Tracking remote branch set
git town sync
# Sync current branch with parent (and all ancestors) ❯ git town sync [main] git fetch origin [main] git rebase origin/main [feature/payment-gateway] git rebase main [feature/payment-gateway] git push --force-with-lease ✓ main is up to date ✓ feature/payment-gateway rebased onto main ✓ Remote branch updated # Sync all branches in the repo ❯ git town sync --all
git town ship
# Ship a feature branch — merge and cleanup ❯ git town ship 1. git fetch origin 2. git rebase origin/main 3. git checkout main 4. git merge --squash feature/payment-gateway 5. git commit 6. git push 7. git push origin :feature/payment-gateway 8. git branch -d feature/payment-gateway ✓ feature/payment-gateway shipped to main ✓ Remote branch deleted ✓ Local branch deleted
git town diff-parent
# Show diff against parent branch (not just origin) ❯ git town diff-parent Diffing feature/payment-gateway against main diff --git a/src/payments/gateway.ts b/src/payments/gateway.ts new file mode 100644 index 0000000..a4f9c21 --- /dev/null +++ b/src/payments/gateway.ts +export class PaymentGateway { + async charge(amount: number) { ... } +} # Useful with stacked branches — see only YOUR changes

Together

Better together,
unstoppable workflow

Worktrees handle the where (parallel directories), Git Town handles the how (automated choreography). Combined, they eliminate every friction point in a modern feature branch workflow.

Git Worktrees
Gives you physical parallelism. Each branch lives in its own directory — your editor, terminal, and tools can all be open simultaneously without collision.
  • No stash/unstash cycles
  • Multiple IDE windows, one codebase
  • Context switching is just cd
  • CI/CD scripts can reference parallel paths
  • Safe for long-running compiles or watchers
Git Town
Gives you workflow automation. The multi-step Git ceremonies for creating, updating, and shipping branches collapse into single commands.
  • Branch creation always starts from latest main
  • Sync keeps branches rebased — clean history
  • Ship cleans up local and remote automatically
  • Stacked branches are first-class citizens
  • Undo support for most commands
Real-world workflow combining both tools
1
Start a new feature
git town hack feature/auth — creates the branch from latest main, sets up tracking. Git Town handles all the fetch-checkout-rebase-push ceremony.
2
Hotfix interrupts — no problem
git worktree add ../hotfix -b hotfix/bug origin/main — spin up a new directory. Your feature work is untouched, running, compiling in its own space.
3
Fix, ship the hotfix
From the hotfix worktree: git town ship — merges to main, deletes the branch, pushes. Zero cleanup steps. Switch back to feature directory with just cd.
4
Keep feature in sync
The hotfix landed on main. git town sync in the feature worktree — rebases the feature branch onto the now-updated main. Clean history, no merge commits.
5
Ship the feature
git town ship from the feature worktree — squash merge, push to main, delete local and remote branch. git worktree remove ../project-feature to clean up the directory.
The takeaway: Worktrees eliminate the physical cost of branch switching. Git Town eliminates the cognitive cost of Git ceremony. Together, they let engineers stay in flow — working on multiple things simultaneously with a clean, consistent workflow.
